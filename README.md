# 🧙 ZX Spectrum Core IDE (Rust + egui) — Промышленная Спецификация Проекта

Интегрированная среда разработки (IDE) нового поколения, созданная на стеке **Rust**, для проектирования, визуализации, валидации и автоматической кроссплатформенной сборки Pixel-Art игр под платформы **ZX Spectrum 48K/128K**. 

IDE полностью заменяет разрозненный и морально устаревший стек консольных утилит (Mappy, Ponedor, msc.exe, sprcnv.exe) единой экосистемой с двусторонней синхронизацией ресурсов, визуальным проектированием коллизий и интерактивным HUD-конструктором. Целевым компилятором выступает **Z88DK**, а движком — **La Churrera (MTE MK1)** от Mojon Twins.

## 📌 1. Архитектурный Стек и Управление Окнами (UI/UX)

### Технические параметры
*   **Язык ядра:** Rust (Стабильная ветка 2021).
*   **Графический UI движок:** `eframe` / `egui` версии `0.25.0`. Активирован feature-флаг `"persistence"` для автоматического сохранения геометрии окон на диск.
*   **Оконный менеджер (Docking):** `egui_dock` версии `=0.10.0`. Активирован флаг `"serde"` для сериализации состояния рабочих зон.
*   **Стиль оформления:** Матовая тема Dark-Neon (Cyberpunk-эстетика). Rounding (скругление) виджетов и вкладок принудительно занулено (`rounding = 0.0`), толщина разделителей окон установлена в `3.0` пикселя.

### Макет интерфейса (Blender / Unity Style Layout)
Интерфейс распределен по трем независимым зонам с жесткой иерархией:
1.  **Постоянная левая док-панель `📁 Проект` (`src/ui/project_tree.rs`):** Полноценное дерево файлов проекта. По двойному клику мыши на узел (`double_clicked()`) генерируется сигнал `CustomTab`. Чтобы избежать конфликтов Borrow Checker, сигнал отправляется во временный потокобезопасный буфер контекста `ui.ctx().data_mut()`. В конце главного цикла `update()` в файле `src/app/mod.rs` сигнал извлекается через `remove_temp` и безопасно сканируется методом `self.dock_state.find_tab(&target_tab)` для мгновенного форсирования фокуса нужного окна.
2.  **Центральное универсальное гнездо вкладок:** Объединяет в одном окне по умолчанию:
    *   `🗺️ Конструктор мира` (`src/app/tab_viewer.rs`) — Главная рабочая область.
    *   `📜 Редактор скриптов` (`src/ui/script_ide.rs`) — Текстовое поле сценариев `.spt` со сниппетами.
    *   `⚙️ Настройки движка` (`src/ui/configurator.rs`) — Слайдеры баланса и визуальный HUD-конструктор.
3.  **Нижний подвал во всю ширину экрана `💻 Логи компиляции`:** Вывод консоли и логов сборщика, который горизонтально подпирает снизу абсолютно все окна разработки.
4.  **Аварийный сброс:** В верхнем меню `📺 Окно -> 🔄 Сбросить макет панелей` полностью очищается и заново собирается структура `DockState`, принудительно возвращая окна к дефолтным пропорциям и перезаписывая кэш настроек OS.

## 💾 2. Структура Единого Проектного Файла (`map/mapa.prj`)

Проект оперирует единым структурированным JSON-сейвом, который сериализуется/десериализуется без потери точности и хранит в себе:
*   `map_w`, `map_h`: Глобальные размеры мира в экранах (поддержка бесшовных карт до 16x16).
*   `scr_inicio`: Индекс стартовой комнаты (0..255), где спавнится игрок при запуске `.tap`.
*   `view_mode`: Перечисление `EngineViewMode` (`SideView` — платформер / `TopView` — вид сверху).
*   `is_128k`: Булево поле активации AY-музыки и банков памяти 128K Spectrum.
*   `tile_behaviours`: Вектор строго на **48 байт**, хранящий маски физических свойств для каждого тайла палитры.
*   `screens`: Хэш-карта `HashMap<String, ScreenData>`, где ключом выступает строка `screen_ID`. Каждая комната хранит:
    *   `tiles_matrix`: Массив из **150 байт** (сетка 15x10 тайлов) со сквозными индексами блоков 0..47.
    *   `enemies`: Динамический вектор структур `Enemy`. Движок Churrera жестко ограничивает пул врагов на экран — **строго до 3 штук**. Благодарю флагу `NO_MAX_ENEMS` пустые слоты не забиваются мусором, а инициализируются типом `0` (отсутствует).
    *   `hotspot`: Структура интерактивного объекта на экране (Тип предмета, координаты X и Y). На один экран может быть назначен **строго 1 хотспот**.

## 🛠️ 3. Подробная Спецификация Реализованных Модулей

### 🧱 3.1. Окно «Конструктор Мира» и Сквозная Физика
*   **Пиксельный х2 Холст:** Размеры тайла Spectrum отрендерены в масштабе 1:2 — **32x32 пикселя**. Общий физический размер интерактивной зоны холста равен **480x320 пикселей** (15 на 10 знакомест тайлов).
*   **Фикс Texture Bleeding:** Текстуры загружаются с Point Filtering фильтрацией без размытия (`TextureOptions::NEAREST`). При расчете UV-координат из атласа применяется микро-отступ `eps = 0.5` пикселя:
    \[\text{uv\_min} = \frac{\text{tile\_x} + \text{eps}}{256.0}, \quad \text{uv\_max} = \frac{\text{tile\_x} + 16.0 - \text{eps}}{256.0}\]
    Это полностью убирает швы и наползания краев соседних блоков, делая рендеринг идеально бесшовным.
*   **Двухэтажный Sidebar инструментов:** Развернут слева от холста. Высота распределена адаптивно по Flex-схеме через `allocate_ui_with_layout`:
    *   **52% высоты — Палитра Тайлов:** Выводит сетку блоков 0..47 по 4 штуки в ряд. Специальные тайлы 16 (Аптечка), 17 (Лут), 18 (Ключ) скрыты из статической палитры, так как вынесены в хотспоты.
    *   **28% высоты — Палитра Спрайтов Врагов:** Автоматически подгружает пиксельные кадры монстров. Клик по любому элементу палитра тайлов или врагов мгновенно переключает кисть холста в Immediate Mode режиме (переключатели убраны с тулбара).
    *   **15% высоты — Универсальный Инспектор:** Открывается при клике на любой тайл палитры. Позволяет настраивать байт физического поведения в Си-массиве: `0` — Walkable (проходимый), `1` — Kills (шипы/лава), `4` — Platform (полупроходимая платформа), `8` — Obstacle (стена). Для тайлов 14..19 здесь же открываются чекбоксы условных макросов (`PLAYER_PUSH_BOXES`, `ACTIVATE_KEYS_AND_LOCKS`).

### 👾 3.2. Конвейер Спрайтсета и Линейных Траекторий Mojon Twins
*   **Интеркаляция масок по горизонтали:** Модуль `image_processor.rs` парсит файл `gfx/sprites.png` (строго 256x32 пикселя) с горизонтальным шагом в 32 пикселя. Он берет спрайт 16x16 и автоматически генерирует для него бинарную маску прозрачности в соседнее правое окно на основе черного цвета `RGB(0,0,0)`. Фон заливается белым `RGB(255,255,255)` (прозрачность), тело объекта — черным `RGB(0,0,0)`. Верхняя строка (Y:0..16) — Игрок (8 кадров), нижняя (Y:16..32) — Враги и Платформы.
*   **Конструктор векторов движения (Типы 1-4):** Для линейных врагов и движущихся платформ (Тип 4 в SideView) в сайдбаре выведены DragValue-счетчики лимитов `X-Мин / X-Макс` и `Y-Мин / Y-Макс`. IDE автоматически определяет ось движения. Изменение лимитов в реальном времени растягивает поверх холста карты цветные векторные линии патрулирования (бирюзовая для оси X, неоново-зеленая для оси Y) с точками разворота ИИ.

### 📺 3.3. Визуальный HUD-Конструктор Знакомест
*   **Сетка 32x24:** Развернута во вкладке «Настройки движка». Моделирует полный экран Spectrum (256x192). Каждая ячейка сетки равна 12 пикселям на экране разработчика.
*   **Viewport Разметка:** Накладывает серую неактивную зону на игровое поле на основе макросов `VIEWPORT_X` и `VIEWPORT_Y`. Размер окна жестко равен 30x20 знакомест.
*   **Drag-and-Drop Интерфейс:** Позволяет ЛКМ перетаскивать плашку здоровья `❤️` строго по сетке знакомест. Параметры `hud_life_x` и `hud_life_y` автоматически сохраняются в формате `u32` в памяти проекта.
*   **Интеллектуальная защита:** Если пользователь перетаскивает HUD-счетчик внутрь зоны Viewport, сверху загорается ярко-красный оверлей предупреждения: *«КОЛЛИЗИЯ: HUD ПЕРЕСЕКАЕТ ИГРОВОЙ ЭКРАН!»*, блокируя невалидную сборку.

### 🔍 3.4. Декомпозированные Си-генераторы ресурсов (`src/core/`)
*   **`validator.rs` (Attribute Clash):** Попиксельно сканирует заставку `gfx/title.png` сеткой 8x8. Если внутри одного знакоместа обнаружено более 2 уникальных цветов — фиксирует коллизию платформы и подсвечивает невалидные зоны на холсте карты полупрозрачным красным оверлеем со сдвигом сетки 16x16.
*   **`exporter_config.rs` (config.h):** Читает файл-заготовку `templates/config.h.template`. Подключает библиотеку `chrono` и выжигает в заголовке текущую живую системную дату компьютера. Генерирует параметры Fixed-Point физики (константы делятся на 64), прописывает координаты HUD и разворачивает массив `tile_behaviours` в плоский Си-массив `unsigned char comportamiento_tiles []` на 48 байт (три строки по 16 чисел).
*   **`exporter_enemies.rs` & `exporter_hotspots.rs` (enems.h):** Пересчитывают координаты сетки холста (0..14) в экранные пиксели Спектрума методом умножения на **16** (нативный шаг тайла в игре). 
    *   Враги упаковываются в массив структур `MALOTE`. Сборщик на основе лимитов вычисляет стартовые векторы хода `mx, my` (1 или 0) и считает глобальные дефайны `N_ENEMS_TYPE_X` и макрос `BADDIES_COUNT` (сумма типов 1+2+3).
    *   Хотспоты сжимаются в один байт `xy` по правилу Mojon Twins:
        \[xy = (y \cdot 16) + x\]
        Если на экране предмета нет — прописывается пустая Си-заглушка `{0, 0, 0}`. Генерирует дефайны `N_HOTSPOTS_TYPE_X`. Оба модуля сливают код в единый результирующий заголовок `dev/enems.h`.
