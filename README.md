# 🧙 ZX Spectrum Core IDE (Rust + egui) — Спецификация Проекта

Интегрированная среда разработки (IDE) промышленного уровня, созданная для проектирования, визуализации и автоматической сборки Pixel-Art игр под платформы **ZX Spectrum 48K/128K** на базе Си-движка **La Churrera (MTE MK1)** от Mojon Twins.

---

## 📌 Архитектурный стек и Технические реалии (Памятка для ИИ)

### 1. Ядро софта и UI-подсистема
*   **Стек разработки:** Язык Rust (Стабильная ветка).
*   **Графический движок UI:** `eframe` / `egui` версии `0.25.0` (Включен флаг `persistence` для жесткого кэширования геометрии окон на диск).
*   **Менеджер интерактивных панелей:** `egui_dock` версии `=0.10.0` (Включен флаг `serde` для полной сериализации стейта рабочих зон).
*   **Улучшение (Автосохранение):** Окна IDE полностью запоминают свое положение, ширину разделителей и активные вкладки между перезапусками. Кэш пишется в локальную базу данных OS.
*   **Стиль оформления:** Dark-Neon матовая тема (Hi-Tech Cyberpunk) со сверхтонкими швами разделителей панелей. Кастомизировано скругление вкладок док-менеджера (`rounding = 0.0`).

### 2. Структура Единого Проектного Файла (`map/mapa.prj`)
Проект оперирует единым структурированным JSON-сейвом, включающим в себя:
*   Глобальные размеры игрового мира в экранах (поддержка карт до 16x16) и индекс стартовой комнаты (`scr_inicio`).
*   Флаги ручной активации геймплейных механик и ролей спец-тайлов (`role_pushbox_active`, `role_lock_active` и т.д.).
*   Полную **матрицу физического поведения тайлов `tile_behaviours` на 48 байт**.
*   Двухслойные данные каждого экрана:
    *   Слой 1: `tiles_matrix` (Плоский массив из 150 байт, кодирующий тайлы 0..47).
    *   Слой 2: `enemies` (Динамический вектор структур `Enemy`, строго до 3 юнитов на экран согласно правилу `NO_MAX_ENEMS`).

## 🔄 Реализованные модули (Текущий прогресс)

### 🧱 1. Графический Холст и Разделенные Палитры (UI/UX)
*   **Pixel-Art Масштабирование x2:** Размер Спектрум-тайла на холсте увеличен с 16x16 до **32x32 пикселей**. Общий физический размер холста одного экрана равен **480x320 пикселей** (15x10 тайлов).
*   **Устранение Texture Bleeding:** Графический контекст VRAM принудительно переведен в режим Point Filtering (`TextureOptions::NEAREST`). При нарезке UV-координат атласа внедрен микро-отступ `eps = 0.5` пикселя. Полосы и наползания краев соседних блоков полностью устранены, холст рендерится бесшовно.
*   **Дискретная Сетка:** Поверх графики на холст наложена тонкая, полупрозрачная векторная сетка с шагом 32 пикселя (`line_segment` с прозрачностью альфа-канала `25`), служащая ориентиром.
*   **Глобальный тулбар управления слоями:** Выбор активного слоя рисования («🧱 Тайлы окружения» / «👾 Враги и Сущности») вынесен на верхний постоянный тулбар. Переключение радио-кнопок на лету подменяет содержимое левого бокового док-окна `TilePalette`.
    *   **В режиме «Тайлы окружения»:** Выводится полная сетка тайлов 0..47, упакованная по 4 элемента в ряд. Клик на любой тайл открывает под сеткой сквозной **Инспектор Физических Масок (0: Walkable, 1: Kills, 4: Platform, 8: Obstacle)** на полные 48 байт. Для спец-тайлов 14..19 открываются дополнительные Си-макросы условной компиляции.
    *   **В режиме «Враги и Сущности»:** Палитра переключается на отображение 5 классов ИИ. Каждая кнопка содержит реальный пиксельный спрайт монстра, вырезанный из видеопамяти.

### 👾 2. Спецификация Спрайтсета и Линейных Траекторий Mojon Twins
*   **Интеркаляция масок по горизонтали:** Модуль `image_processor` парсит `gfx/sprites.png` (256x32) пошагово по горизонтали (шаг 32 пикселя), считывая спрайт 16x16 и автоматически генерируя для него бинарную маску прозрачности в соседнее правое окно на основе черного цвета `RGB(0,0,0)`. Верхняя строка (Y:0) отдана под кадры игрока, нижняя (Y:16) — под врагов и платформы.
*   **Конструктор векторов движения (Типы 1-4):** Для линейных врагов и движущихся лифтовых платформ (Тип 4 в режиме SideView) реализовано автоматическое распознавание оси хода (`↔️ Горизонтально` / `↕️ Вертикально`). В боковой панели выведены DragValue-счетчики лимитов `X-Мин / X-Макс` и `Y-Мин / Y-Макс`.
*   **Интерактивный оверлей:** Изменение лимитов движения в инспекторе в реальном времени растягивает на холсте карты цветные пунктирные линии патрулирования (бирюзовая для горизонтали, неоново-зеленая для вертикали) с маркерами точек разворота ИИ.

### 🔍 3. Системные валидаторы (Attribute Clash & Шаблоны)
*   **Сканер знакомест:** Модуль `validator` считывает заставку `gfx/title.png` и проверяет ее сеткой 8x8 пикселей. Зоны, нарушающие ограничения платформы (более 2 цветов на знакоместо), подсвечиваются на холсте карты **полупрозрачным красным оверлеем** со сдвигом 16x16.
*   **Генератор по эталону (`dev/config.h`):** Внедрен полный, эталонный текстовый шаблон `templates/config.h.template`, содержащий абсолютно все макросы условной компиляции, Fixed-Point физику (константы, деленные на 64) и экранные координаты перетаскиваемого HUD. При экспорте Rust формирует чистый Си-файл с раскомментированием нужных директив и разворачивает Си-массив `comportamiento_tiles` на 48 байт.

## 📂 Структура каталогов проекта на диске

```text
zx_project_root/
├── Cargo.toml               # Конфиг Cargo (eframe = "0.25.0" + persistence, egui_dock = "=0.10.0" + serde)
├── templates/               # ШАБЛОНЫ СИ-КОДА
│   └── config.h.template    # Эталонный шаблон со всеми макросами условной компиляции Mojon Twins
├── map/                     # ПРОЕКТНЫЕ ДАННЫЕ И JSON СЕЙВЫ
│   └── mapa.prj             # Единый файл проекта (Карта, Баланс, 48-байтная физика тайлов, Враги)
├── dev/                     # АВТОГЕНЕРАЦИЯ (Сюда пишет Rust ядро)
│   └── config.h             # Сгенерированные директивы препроцессора Си и массив поведения
├── gfx/                     # ИГРОВЫЕ ГРАФИЧЕСКИЕ АССЕТЫ (PNG)
│   ├── work.png             # [Входной] Исходный тайлсет 256x48
│   ├── mappy.png            # [Авто] Тайлсет с зануленным тайлом 0 для слоев движка
│   ├── sprites.png          # [Входной/Авто] Интеркалированный спрайтсет 256x32 с собранными масками
│   └── title.png            # [Входной] Экран заставки для валидатора Attribute Clash
└── src/                     # ИСХОДНЫЙ КОД IDE НА RUST
    ├── main.rs              # Точка входа, инициализация NativeOptions, создание системных папок
    ├── models/              # Модели данных (project.rs, screen.rs, config.rs)
    ├── core/                # Системное ядро (io.rs — шаблонизатор, validator.rs, image_processor.rs)
    ├── ui/                  # Слой представлений (map_canvas.rs, palette_tiles.rs, palette_enemies.rs, ...)
    └── app/                 # Декомпозированный модуль интерфейса приложения
        ├── mod.rs           # Корневой диспетчер eframe, темы оформления и структуры
        ├── states.rs        # Перечисления стейтов (Tab, CustomTab, MapEditMode, WizardStep)
        ├── wizard.rs        # Мастер пошаговой инициализации параметров новой игры
        ├── menu_bar.rs      # Виджет выпадающего меню (Сохранить / Выход)
        └── toolbar.rs       # Виджет глобального тулбара (Импорт графики / Переключатель слоев холста)
```

## 🛠️ Предстоящие задачи (Вектор дальнейшей разработки)

1.  **Сборщик бинарных структур сущностей (`dev/enems.h`):** Написать модуль трансляции, который будет забирать из JSON-памяти проекта расставленных врагов, их типы, точки спавна и настроенные границы траекторий `x1, x2, y1, y2`, упаковывая их в плоский байтовый массив структуры `.ene` для чтения Си-движком при смене экранов.
2.  **Модуль автоматизации `compiler_runner.rs`:** Реализовать кроссплатформенный запуск компилятора через `std::process::Command`. Скрипт будет брать нативный Си-компилятор **Z88DK** из изолированных папок `bin/windows/`, `bin/linux/`, `bin/macos/` и одной кнопкой в IDE собирать готовую игру в контейнер `output/game.tap` для запуска в эмуляторе.
