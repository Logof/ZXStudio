📑 Архитектура и структура проекта ZXStudioДанный документ содержит детальное описание файловой структуры интегрированной среды разработки (IDE) ZXStudio, написанной на языке Rust с использованием графической библиотеки egui / eframe и ориентированной на экосистему ZX Spectrum (MTE MK1 v4.8 "Zombie").🌲 Дерево проекта и описание модулей📁 Корневая директория (Root)assets/ — Встроенные статические бинарные ресурсы IDE.churrera_v48_template.zip — Полный оригинальный ZIP-архив Си-ядра и ASM-библиотек фреймворка La Churrera v4.8. Вшивается намертво в исполняемый файл с помощью макроса include_bytes! и автоматически разворачивается в корень папки разработчика при создании новой игры.churromain.c — Шаблон главного Си-файла для многоуровневых проектов.ramiro3v3.tap — Эталонный тестовый контейнер ленты.Cargo.toml / Cargo.lock — Манифест зависимостей Cargo (egui, eframe, serde, zip, encoding_rs).locales/ — Файлы внешней статической локализации интерфейса (ru.json, en.json).templates/ — Шаблоны метакода.config.h.template — Главный эталонный заголовочный файл с текстовыми маркерами-плейсхолдерами (например, {{TOGGLE_COMPRESSION}}, {{PLAYER_G}}), в которые генератор ядра подставляет значения из ОЗУ.📁 Папка исходного кода src/🔹 main.rsТочка входа в приложение Rust. Инициализирует контекст графического движка eframe, подгружает сохраненные системные настройки из хранилища cc.storage (пути к компиляторам, хоткеи, историю проектов) и запускает главный оконный цикл.📁 src/models/ — Слой абстракции данных (Data Models)Модели данных, сериализуемые в JSON (формат сохранения проекта .prj) с поддержкой миграций старых версий.mod.rs — Координатор модулей моделей.project.rs — Главная структура ProjectData. Описывает текущий выбранный уровень, ОЗУ шрифта (768 байт), а также глобальные фичи ветки v4.8: режим упаковщика ресурсов (CompressionMode::Zx0/Zx7/Appack) и флаги Z80 ASM-оптимизаций.screen.rs — Структура ScreenData. Содержит массив tiles_matrix (150 байт знакомест комнаты), а также массивы привязанных к экрану врагов и хотспотов.enemy_ai.rs — Перечисления конфигурации искусственного интеллекта ретро-сущностей.📁 config/ — Структурированные настройки физики и HUD.general.rs — Флаги архитектуры (128K режим, кастомный декодер карты).movement_controls.rs — Константы физики Fixed-Point (гравитация, скорость, Jetpac) и проекция камеры (Top-View / Side View).player_physics.rs — Триггеры поведения игрока (отскоки от стен, мерцание).shooting_boxes.rs — Настройки стрельбы, разрушаемых блоков и толкания ящиков.mechanics_enemies.rs — Логика спавна, коллизий и сбора предметов.hud_rendering.rs — Экранные текстовые координаты элементов HUD и счетчиков игрока.map_goals.rs — Размеры сетки комнат мира (ширина/высота карты в экранах).tile_behaviour.rs — Массив поведений тайлов (твердый, блок, смерть, вода).📁 src/ui/ — Графический интерфейс пользователя (GUI)Рендеринг интерактивных элементов управления и вкладок среды разработки.mod.rs — Диспетчеризация отрисовки интерфейса.project_tree.rs — Левое дерево файлов проекта (просмотр ресурсов gfx/, map/, script/). Поддерживает ленивую загрузку скриптов по двойному щелчку мыши.📁 configurator/ — Панель настройки макросов движка La Churrera.mod.rs — Сайдбар меню выбора разделов настроек.font_editor.rs — Пиксельная сетка для ручного рисования 8x8 шрифта игры.📁 general/ — Модульный каталог общих настроек, адаптированный под ветку v4.8.mod.rs — Координатор подмодулей.level_management.rs — Добавление/удаление уровней (Multilevel).platform_settings.rs — Выбор разрешения графической архитектуры (TileMode).compression_asm.rs — Чекбоксы переключения упаковщиков ZX7 / ZX0 и ассемблерных оптимизаций спрайтов.audio_system.rs — Интеграция плееров трекерной музыки Wyz / Arkos Tracker.rendering_settings.rs — Ограничители частоты кадров движка (FPS Limiter).(Остальные файлы player_physics.rs, tile_behaviour.rs и т.д.) — Выводят ползунки и переключатели под соответствующие структуры из слоя models/config/.📁 map_editor/ — Полноценный 2D-редактор игрового мира.📁 left_panel/ — Палитры выбора тайлов (tiles_palette.rs) и типов врагов (enemies_palette.rs).📁 map_canvas/ — Графический холст комнат с поддержкой камеры (camera.rs), скроллинга (scroll.rs), послойного рисования блоков (tile_layer.rs) и расстановки врагов методом Drag-and-Drop (entity_layer.rs).📁 script_ide/ — Текстовый редактор сценариев Churscript (editor.rs) с базой готовых сниппетов (snippets.rs).📁 hud_editor/ — Визуальный конструктор разметки знакомест интерфейса игрока (canvas.rs, sidebar.rs).📁 src/app/ — Главное ядро координации IDE (Application Core)Верхнеуровневые сервисы, оркестрирующие работу оконных систем и внешних потоков.mod.rs — Оконный диспетчер, реализующий трейт eframe::App.app_struct.rs — Определение главной структуры ZxIdeApp, хранящей текстуры, буфер логов и каналы связи.theme.rs — Матовые темно-серые неоновые стили оформления IDE (Cyberpunk Modern Dark).toolbar.rs — Верхняя панель быстрого доступа. Содержит кнопки сохранения JSON, экспорта ресурсов и асинхронную кнопку 🚀 Собрать game.tap [INDEX].dock_layout.rs — Сетка геометрии окон egui_dock (Project Tree слева, Консоль снизу, Холсты по центру).tab_viewer.rs — Реализация трейта TabViewer для динамического переключения контента вкладок. Модернизирована: вкладка Console выводит логи через выделяемое и копируемое поле egui::TextEdit.world_inspector.rs — Фоновый аудит мира. На холст карты выводит исключительно ошибки застревания сущностей, отсекая графические коллизии.compiler_observer.rs — Неблокирующий опрос канала логов сборщика (try_recv). Ведет учет временных меток ([ЧЧ:ММ:СС]) и чистит старые строки при превышении лимита в 500 строк во избежание утечек памяти.signal_handler.rs — Мощная шина реактивных UI-сигналов. Отлавливает триггер сборки, клонирует ОЗУ проекта и запускает асинхронный фоновый поток компиляции [INDEX].📁 wizard/ — Пошаговый стартовый мастер создания новой игры (welcome_choice.rs, select_platform.rs, name_and_path.rs).📁 src/core/ — Среда выполнения, утилиты и компиляторы (Compiler Runtime)Низкоуровневый движок нативной обработки данных, заменяющий оригинальные CLI-утилиты Mojon Twins.mod.rs — Регистрация модулей бэкэнда.io.rs / validator.rs — Файловый ввод-вывод и проверка валидности.image_processor.rs — Модуль попиксельной сквозной трансляции и сборки текстурных атласов.project_template.rs — Нативный распаковщик встроенного ZIP-архива движка (на базе библиотеки zip).compressors.rs — Кроссплатформенный шелл-раннер упаковщиков. На Linux, Windows и macOS запускает нативные бинарники zx7 / zx0 Эйнара Саукаса, перехватывает ошибки и логирует степень сжатия данных в байтах.📁 utils/ — Встроенные графические конвертеры (чистый Rust).ts2bin.rs — Нативная замена ts2bin.exe. Нарезает PNG-картинку на тайлы 8x8, кодирует маски пикселей, подмешивает 768 байт шрифта из памяти проекта и выпекает готовый бинарник tileset.bin [INDEX].png2scr.rs — Нативная замена png2scr.exe. Превращает 256x192 PNG в полноценный дамп видеопамяти Спектрума (6912 байт) с учетом интерлиньяжной адресации экранов ZX Spectrum [INDEX].📁 validator/ — Сервисы глубокого анализа ресурсов.attribute_clash.rs — Проверка заставок на конфликты наложения цвета в рамках знакомест 8x8 [INDEX].world_collisions.rs — Проверка застревания врагов в стенах комнат.📁 exporter/ — Модули экспорта игровых массивов.exporter_map.rs / exporter_enemies.rs / exporter_hotspots.rs — Выгрузка Си-массивов карты, врагов и хотспотов.📁 exporter_config/ — Новая модульная структура генератора настроек ядра.mod.rs — Главный координатор сборки dev/config.h.platform_macros.rs — Генерирует дефайны сжатия (COMPRESSION_ZX0) и Z80 ASM-оптимизаций под спецификацию v4.8 [INDEX].mechanics_physics.rs — Выгрузка макросов игровых триггеров и боксов стрельбы.hud_macros.rs — Выгрузка статических экранных координат HUD.movement_physics.rs — Выгрузка математических Fixed-Point констант перемещений.📁 pipeline/ — Устаревший синхронный CLI-конвейер сборки ассетов (main_task.rs, gfx_task.rs [исправлен], sprite_task.rs [исправлен]).📁 compiler_runner/ — Новый асинхронный кроссплатформенный оркестратор компиляции (Этап 4). Выполняет пошаговую сборку игры в фоновом изолированном потоке ОС [INDEX].mod.rs — Главный фоновый оркестратор потока сборки.step1.rs — Нативный парсер сценариев Churscript. Нарезает текст по секциям LEVEL X и токенизирует в байт-коды dev/mscX.h. Интегрирована библиотека encoding_rs для поддержки файлов в кодировке Windows-1251 (ANSI) с кириллическими комментариями.step2.rs — Интеллектуальный сборщик. На лету проверяет наличие исходников, при их отсутствии вызывает project_template [INDEX]; если нет бинарной карты — нативно собирает её из ОЗУ в map/mapa.bin [INDEX]; затем вызывает ts2bin / png2scr для сборки заставок и тайлсетов [INDEX]; на лету исправляет устаревший ассемблерный синтаксис в файле dev/mainloop.h [INDEX] и запускает Си-компилятор Z88DK (zcc) с флагами -lsplib2 -zorg=24200 [INDEX], автоматически линкуя локальные библиотеки проекта [INDEX].step3.rs — Нативный токенизатор Sinclair BASIC. Генерирует токенизированное тело программы лоадера (команды CLEAR, LOAD "" CODE, RANDOMIZE USR) и собирает файл загрузчика dev/loader.tap с расчетом XOR-контрольных сумм без внешних утилит [INDEX].step4.rs — Сканирование и подготовка бинарных плейсхолдеров экранов заставок.step5.rs — Финальный контейнерный монтаж релиза. Побайтово склеивает три TAP-блока (BASIC-лоадер со строки 10 [INDEX], блок заставки по адресу 32768 [INDEX] и машинный код игры по адресу 24200 [INDEX]), рассчитывая контрольные суммы блоков [INDEX], и выпекает готовый эмуляторный образ вида Имя_Игры.tap в корень папки проекта [INDEX].⚙️ Взаимодействие компонентов при асинхронной сборке[Пользователь кликает "Собрать game.tap"]
